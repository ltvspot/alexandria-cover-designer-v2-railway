## Project Overview

Alexandria Cover Designer v2 is a single-page application (SPA) for generating AI-illustrated book covers for a classical fiction library. It:

- **Syncs** a book catalog from Google Drive (folder-based, each subfolder = one book with a cover JPG)
- **Generates** circular medallion-style illustrations via the OpenRouter API (15 AI models supported)
- **Composites** those illustrations into the medallion area of existing cover templates using a canvas-based pipeline with a pre-computed alpha mask
- **Scores** image quality using a 7-factor client-side algorithm optimized for circular vignette compositions
- **Reviews** and approves winner images per book, downloadable as a ZIP
- **Tracks** costs, quality, and generation history with analytics charts
- **Persists** settings server-side via a Python CGI endpoint so API keys survive page reloads

The app runs entirely in the browser with no build step. All data storage is in-memory (JS objects mimicking IndexedDB), with settings written to disk via the CGI endpoint.

**Medallion Detection Algorithm:**
- Scales analysis to 300├Ś300px canvas, focusing on the right 55% of the cover (the front panel)

- Returns null if best score < 8 (no clear boundary)
- Maps detected coords back to full-image space
- Sanity check: cx must be 1500ŌĆō3500, cy must be 500ŌĆō2200, radius must be 200ŌĆō900 (front-panel zone)

### 7. `js/compositor.js`
**Lines:** 206

**Purpose:** Canvas-based compositing engine (v6). Places an AI-generated illustration into the circular medallion area of a book cover template using a two-layer "overlay-on-top" approach with a pre-computed alpha mask PNG.

**Architecture (Overlay-on-Top, v6):**
1. **Layer 0 (bottom):** AI illustration, smart-cropped to fill the medallion area with 15% overflow (`fillRadius = radius * 1.15`), clipped to a circle
2. **Layer 1 (top):** Original cover image with alpha mask applied using `destination-in` compositing ŌĆö the mask (white = opaque, black = transparent) makes the medallion center transparent so the illustration shows through, while all frame/text/background pixels remain pixel-perfect

**Mask:** Loaded once from `img/medallion_mask.png` and cached in `_maskImagePromise`.

**Default Medallion Position:**
- `MEDALLION_CX = 2850`, `MEDALLION_CY = 1350`, `MEDALLION_RADIUS = 520`
- Cover dimensions: 3784├Ś2777px (wraparound ŌĆö back + spine + front)

**Exports (`window.Compositor`):**

| Export | Description |
|--------|-------------|
| `smartComposite(coverImg, generatedImg, cx, cy, radius)` | Main compositing pipeline; returns `HTMLCanvasElement` |
| `compositeOnCover(coverImg, generatedImg, cx, cy, radius, feather)` | Backward-compat alias ŌåÆ calls `smartComposite` |
| `findBestCropCenter(imageElement)` | Energy-based detail center detection; returns `{ x, y }` in 0ŌĆō1 normalized coords |
| `createThumbnail(canvas, maxWidth=400)` | Scales canvas to a thumbnail |
| `canvasToBlob(canvas, type, quality)` | Returns `Promise<Blob>` |
| `canvasToDataUrl(canvas, type, quality)` | Returns data URL string |
| `loadMask()` | Loads and caches the alpha mask |
| `MEDALLION_CX`, `MEDALLION_CY`, `MEDALLION_RADIUS` | Constants |

**`findBestCropCenter` Algorithm:**
- Renders image at 150├Ś150
- Computes Sobel-like gradient energy per pixel
- Applies 3├Ś3 Gaussian blur to the energy map
- Computes weighted centroid (energy-weighted x, y average)
- Returns centroid in [0,1] normalized coords, clamped to [0.2, 0.8]

### 8. `js/quality.js`
**Lines:** 346

**Purpose:** Client-side image quality scoring optimized for circular medallion illustrations. Provides a composite score (0ŌĆō1) using 7 sub-scores, plus a detailed breakdown for debugging.

**Exports (`window.Quality`):**

| Export | Description |
|--------|-------------|
| `scoreImage(canvas)` | Legacy API: scores a canvas using 4 color-based factors |
| `scoreGeneratedImage(imageElement)` | Primary API: full 7-factor scoring |
| `getDetailedScores(imageElement)` | Returns `{ overall, edgeContent, centerOfMass, circularComposition, color, brightness, contrast, diversity }` with per-score `{ score, weight }` |

**Scoring Formula (`scoreGeneratedImage`):**

| Sub-Score | Weight | Algorithm |
|-----------|--------|-----------|
| Edge Content Penalty | 25% | Outer 15% ring avg gradient vs inner 50% avg gradient ŌĆö high edge detail at boundary = bad crop |
| Center of Mass | 20% | Brightness+saturation-weighted centroid should be near geometric center |
| Circular Composition | 20% | Brightness stddev inside central 70% circle should exceed stddev in corners |
| Color Variance | 12% | RGB channel variance should be high (not solid color) |
| Brightness | 8% | Optimal target brightness ~45%; penalty for very dark or very bright |
| Contrast | 8% | Standard deviation of brightness should be Ōēź 0.25 |
| Color Diversity | 7% | Spread between R, G, B channel averages |

**Analysis Resolutions:** Edge content at 200├Ś200; center of mass and circular composition at 150├Ś150; legacy color scores at 300├Ś300 (max).

**Dependencies:** None (standalone canvas operations)

### Navigation & Routing System

The app uses **hash-based routing** ŌĆö no server-side routing required. All navigation is through `location.hash`.

**Router Flow:**
1. `app.js` defines a `PAGES` map: `{ pageName: { title: string, render: () => Pages.pageName.render() } }`
2. `window.addEventListener('hashchange', renderPage)` is the core listener
3. `renderPage()` reads `getPageFromHash()` (strips `#`, splits on `?`, returns page name)
4. Sets `document.getElementById('pageTitle').textContent`
5. Updates `.nav-link.active` classes by matching `link.dataset.page === page`
6. Shows a spinner in `#content`, then calls `config.render()` inside a try/catch
7. On unknown hash: redirects to `#iterate`

**Page Registration Pattern:**
Each page file sets `window.Pages = window.Pages || {}` and then assigns `window.Pages.pageName = { async render() { ... }, ...methods }`. The `window.Pages` namespace is pre-declared in `app.js` so load order within modules doesn't matter.

### Settings Persistence Architecture

Settings have a **three-layer persistence chain**:

```
Browser memory (JS _stores.settings)
    ŌåĢ (on every setSetting call, debounced 300ms)
Server disk (settings_store.json)
    ŌåĢ (via HTTP GET on DB.openDB ŌåÆ initDefaults)
```

**Write path:** Any `DB.setSetting(key, value)` call:
1. Updates `_stores.settings[key] = { key, value }` immediately (in-memory, instant)
2. Schedules `_persistSettings()` with `setTimeout(300ms)`
3. `_persistSettings()` POSTs flat `{ key: value }` JSON to `__CGI_BIN__/settings.py`
4. `settings.py` merges into `settings_store.json` (never replaces, only merges)

**Read path on startup:** `DB.initDefaults()`:
1. `_loadServerSettings()` ŌĆö GETs `__CGI_BIN__/settings.py`, populates `_stores.settings` from server JSON
2. For each key in `defaults`: only sets it if not already present (never overwrites user values)

**Reset path:** Settings page "Reset to Defaults" button:
1. `DB.dbClear('settings')` ŌĆö wipes in-memory store
2. POST to `__CGI_BIN__/settings.py/reset` ŌĆö deletes `settings_store.json`
3. `DB.initDefaults()` ŌĆö re-populates from hardcoded defaults

**CGI URL placeholder:** Both `db.js` and `drive.js` reference `'__CGI_BIN__/settings.py'` and `'__CGI_BIN__/catalog.py'` respectively. This placeholder string must be resolved by the web server or a deploy script to the actual CGI path.

### Job Queue System

The `JobQueue` in `app.js` is a hardened parallel job processor with up to **5 concurrent jobs**.

**State machine per job:**
```
queued ŌåÆ downloading_cover ŌåÆ generating ŌåÆ [retrying] ŌåÆ scoring ŌåÆ compositing ŌåÆ completed
                                    Ōåō                                              Ōåō
                                  failed ŌåÉŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆöŌĆö
```

**Parallel execution:**
- `_fillSlots()` runs while `running.size < MAX_CONCURRENT && queue.length > 0`
- Each slot gets its own `AbortController`
- Jobs are launched with fire-and-forget `.then()` chaining (no `await` in `_fillSlots`)

**Heartbeat (every 1s via `setInterval`):**
- Updates `job._elapsed` (seconds since start) on all running jobs
- Checks for dead jobs (`elapsed > DEAD_JOB_TIMEOUT = 3min`) and force-fails them
- Calls `notify()` ŌåÆ all registered `onChange` listeners fire (drives live UI updates)

**Retry logic:**
- After each generation attempt, quality score is compared to `RETRY_THRESHOLD = 0.35`
- If score is below threshold and attempts remain (up to `MAX_RETRIES = 2`), retry with strengthened circular vignette instruction appended to prompt
- Best result across all attempts is kept regardless
- 429 rate-limit errors do NOT consume a retry slot ŌĆö the current attempt is retried after backoff (5s, 10s, 15s... up to 30s)

**Abort/cancel:**
- `abortJob(jobId)` calls `entry.abortController.abort()` and marks job `failed`
- The abort signal is forwarded through both `OpenRouter.generateImage()` and `Drive.downloadCoverWithRetry()` (via abort event listener)
- `cancelAll()` aborts all running jobs and clears the queue

**Stuck job recovery (E16):**
- On every `init()`, `resumeStuckJobs()` scans all jobs in DB
- Any job with status not in `['completed', 'failed', 'queued']` was interrupted mid-run
- These are marked `failed` with error `'Interrupted by page reload'`

**Cover caching (B6):**
- `CoverCache` holds up to 20 recently downloaded covers (LRU eviction)
- Concurrent jobs for the same book share a single in-flight download via `_pending` Map of Promises
- Includes medallion position detection on first load; subsequent jobs for the same book reuse the detected position

**Cost tracking:**
- `job.cost_usd` accumulates `OpenRouter.MODEL_COSTS[job.model]` for each generation attempt (retries add cost)
- On completion, a `cost_ledger` entry is written
- `updateHeader()` sums `cost_ledger` + in-flight `running` job costs for real-time header display
